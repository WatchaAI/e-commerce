## 这玩意儿是干嘛的

没学过设计、不会 PS、不知道怎么给 AI 写提示词？

**不用想了。** 直接套这个 Skill，三步出图。

灵感来自一个电商详情页工作流，我测了 2 个早上。工作流本身不复杂，但打包成 Skill 绕了点路。思路就三步：

### ① 丢产品图 → AI 自动修瑕疵、合多视图

把你手里的产品照片扔过来，实拍图最好。AI 会自动分析图片瑕疵（模糊、偏色、反光、变形……），生成定制精修提示词，产出白底精修商品图。多角度图还会检查一致性，不会出现「这张图金色那张图银色」的翻车。

### ② 填产品信息、卖点

品类、品牌、卖点、规格、目标人群、使用场景……把你知道的填进去。不知道的空着，AI 不会瞎编。

### ③ 丢参考详情页 → 结合产品图和卖点，直接生成

找一张你觉得好看的详情页长图扔过来，AI 会参考它的结构、节奏、排版风格，结合你的产品图和卖点，直接生成一套完整详情页。

**不用从头想设计，抄参考结构就行。重点是快。**

## 适合谁用

- 电商卖家 / 运营，想快速出详情页但不会设计
- 自媒体博主，需要产品图文展示
- 任何「有产品、没设计团队」的人

覆盖品类：手表、美妆、服饰、鞋包、珠宝、家居、食品、3C 数码、家电……啥都行。

## 怎么用

### 在 Codex 里用（推荐）

1. 把这个文件夹放到 Codex 的 skills 目录下
2. 在 Codex 里跟 AI 说：「帮我做个电商详情页」
3. 按 AI 引导，依次发：产品图 → 产品信息 → 参考详情页
4. 坐等出图

> 目前测下来 **Codex 体验最舒服**。其他能生图的 Agent 也能用，但 Codex 的流程最顺。

### 在自己项目里用

直接把 `ecommerce-detail-page-workflow/` 整个目录集成到你的 Agent 项目里，SKILL.md 就是完整的工作流说明。

## 修改建议

整个 Skill 的主要输入就在于上面那三步：

- 产品图不满意 → 换图重来
- 卖点不对 → 改信息再跑
- 参考页风格不对味 → 换张参考页

每个环节出问题都可以第一时间沟通修改，不用从头重来。
